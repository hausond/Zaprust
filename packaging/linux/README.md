# Linux packaging — элевация (polkit)

## Как Zaprust поднимает права (L4)

Привилегированные операции (правила фаервола, служба systemd, замена ядра)
делаются **реинвоком самого приложения от root через `pkexec`**:

```
pkexec <путь-к-zaprust> --svc <install|start|stop|remove|uninstall> …
pkexec <путь-к-zaprust> --autoselect <gf> [known-good]
pkexec <путь-к-zaprust> --apply-update <zip> <tag> …
```

`pkexec` (часть polkit) показывает GUI-диалог аутентификации и запускает команду
от root. Реинвок исполняется от root, пишет итог операции в `/tmp/zaprust_result.json`
(и `/tmp/zaprust_svc.log`), а неэлевированный GUI читает их **после** завершения
процесса — тот же файловый хэндшейк, что и на Windows.

Окружение реинвоку **не прокидывается** (pkexec его вычищает) — всё нужное
передаётся аргументами. Поэтому межпроцессные файлы-хэндшейки закреплены в `/tmp`
(см. `main::handshake_dir`): после очистки окружения реинвок видит дефолтный
`/tmp`, и GUI смотрит туда же независимо от `$TMPDIR`.

### Коды возврата pkexec

| rc | значение | как трактуем |
|----|----------|--------------|
| 126 | не авторизован / диалог отменён | операция отменена, статус не меняем |
| 127 | pkexec не смог запустить | ошибка запуска |
| иначе | код выхода реинвока | результат + `/tmp/zaprust_result.json` |
| (сигнал) | реинвок убит | умер без result-файла |

Если `pkexec` вообще нет в системе (polkit не установлен) — отдельная ошибка
«pkexec недоступен».

## Опциональная polkit-policy: `dev.zaprust.policy`

Делает диалог внятным («Управление обходом DPI») вместо обобщённого
«run a program as another user».

- **Нужна только при системной установке** в фиксированный путь (distro-пакет).
  Кладётся в `/usr/share/polkit-1/actions/dev.zaprust.policy`. Перед установкой
  подставьте реальный путь бинаря в
  `<annotate key="org.freedesktop.policykit.exec.path">`.
- **Для AppImage / портабла НЕ ставится:** путь динамический (AppImage монтируется
  в `/tmp/.mount_*`), привязать его к фиксированному action нельзя, а заводить
  системный файл при «скачал-и-запустил» неуместно. Там работает дефолтный
  `pkexec` (`org.freedesktop.policykit.exec`) — без своей policy, просто диалог
  с обобщённой формулировкой.

Так компромисс «внятный диалог ↔ совместимость с портабл-поставкой» решается
в пользу совместимости: policy — бонус для пакетов, не требование.
