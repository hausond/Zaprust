# Zaprust — отступления от стандартных step-промптов

Документ фиксирует все места, где итоговая реализация **разошлась** с тем, что описано в `step-01…step-09`. Часть отступлений вынужденные (реальность Windows/Flowseal), часть — по твоим прямым запросам в ходе работы. Для каждого пункта: **что в промпте → что сделали → почему**.

---

## 0. Окружение сборки (вне промптов, но важно)

Промпты подразумевают «`cargo new` и поехали». На деле на машине **не было ни Rust, ни компилятора**.

- **Toolchain:** выбрали **GNU** (`stable-x86_64-pc-windows-gnu`), а не MSVC — чтобы не тянуть многогигабайтные VS Build Tools.
- **Линкер:** rustup-овский self-contained mingw неполный. Доставили портативный **w64devkit** (`C:\hausond\w64devkit`).
- **Костыль `libgcc_eh.a`:** GCC 16 у w64devkit слил `libgcc_eh` в `libgcc.a`, а Rust всё ещё требует `-lgcc_eh`. Создали **пустой** `libgcc_eh.a`-заглушку.
- Для сборки нужно в PATH: `~\.cargo\bin` + `C:\hausond\w64devkit\bin`. Рантайм-зависимостей от w64devkit у бинаря нет (libgcc/pthread линкуются статически) — exe запускается на чистой машине.

> Влияние на шаг 9 (упаковка): дополнительные DLL в релиз класть не нужно.

## 1. Каркас UI — позже сильно упрощён (по твоему запросу)

Промпт (шаг 1) описывает макет с **тремя чипами статуса** (`winws.exe`, `WinDivert`, `Служба`) и декоративными значками окна.

**Отступление (минимализм):**
- Убрали тройку чипов целиком. Оставили **один статус-пилюлю «Работает / Выключен»** в шапке.
- Убрали фейковые кнопки окна (✕ / —) и тех-подпись под дропдауном.
- Кнопку Старт/Стоп растянули **на всю ширину** (раньше жалась влево).
- Лог-бокс тянется на всю оставшуюся высоту.

**Почему:** просил больше минимализма и адаптации по ширине.

## 2. Парсер стратегий — без отступлений по сути

Шаг 2 реализован как в промпте (терпимый парсер `general*.bat`, группировка, абсолютные пути). Нюанс, всплывший позже: реальные батники Flowseal используют переменные `%GameFilterTCP/UDP%`, которые задаёт `service.bat`, — парсер их **намеренно оставляет** как есть, а разрешает уже при запуске (см. п.3).

## 3. Запуск winws — две вынужденные правки

**3.1． Права администратора (ошибка 740).**
Промпт говорит «запускай среду разработки от админа». На практике `winws.exe` требует элевации, и обычный `CreateProcess` падает с **os error 740**. Добавляли детект прав + баннер-предупреждение (позже всё это убрано вместе с прямым запуском — см. п.6-bis).

**3.2． Game Filter резолвится при запуске.**
Не описано в промпте: значения `%GameFilterTCP/UDP/GameFilter%` подставляются в момент старта по тумблеру **Game Filter** (выкл → `12`, вкл → `1024-65535`) — ровно как это делает `service.bat` у Flowseal.

---

## 4. Живой лог winws — РЕАЛИЗОВАН, потом УБРАН

Шаг 4 = живой stdout winws через пайпы + потоки-читатели. Это было сделано и работало.

**Отступление:** после перехода на **service-only** модель (п.6-bis) живой лог самого winws **удалён** — служба запускает winws как отдельный процесс, его stdout не подцепить пайпом. В лог-боксе теперь только события приложения/службы/теста/обновления.

> То есть критерий шага 4 («видно строки запуска winws») в финале **не выполняется** — сознательный размен на простоту service-only.

---

## 5. Детект WinDivert + Тест — выбор TLS-бэкенда

Шаг 5 как в промпте, с уточнениями:
- **TLS для кнопки «Тест»:** взяли `ureq` + **native-tls (Windows SChannel)**, а не rustls/ring — чтобы не воевать с компиляцией `ring` на gnu-toolchain. Голый TCP не годится (DPI режет по SNI), поэтому именно полноценный HTTPS-хендшейк.
- **`targets.txt`:** парсим реальный формат Flowseal `Имя = "https://host"` / `"PING:ip"` (а не просто список доменов). PING-записи проверяем TCP-коннектом на :443.

---

## 6. Служба и автозапуск — метод Flowseal вместо .bat-обёртки

Шаг 6, пункт 1, предлагает **генерировать `.bat`-обёртку** и прописывать её в `binPath` службы (из-за лимита длины аргументов).

**Отступление:** пошли как **реально сделано у Flowseal** — winws прописывается **напрямую** в `binPath` со всеми аргументами инлайн:
```
sc create zapret binPath= "\"…\winws.exe\" <args>" start= auto
```
- Аргументы ALT-стратегий ~4–5 тыс. символов — спокойно влезают в лимит ~8191. Длину проверяем, при превышении — отказ с понятной ошибкой.
- Это совпадает с глобальным правилом проекта «winws не через .bat» (батник-обёртку как службу всё равно нельзя корректно убить).
- Дополнительно: `sc sdset` (изначально — чтобы не-админ мог start/stop; в финальной модели уже не критично), `netsh … timestamps=enabled`, метка стратегии в реестр.

**Модель UAC:** выбрали рекомендованный вариант «по требованию» (элевация отдельной операции через `ShellExecuteExW("runas")`, реинвок самого exe в режиме `--svc`).

---

## 6-bis. ГЛАВНОЕ архитектурное отступление — SERVICE-ONLY (по твоему запросу)

Это самое крупное расхождение со всеми промптами.

Промпты (шаги 3 и 6) подразумевают **два режима**: прямой процесс (ручной лаунчер) **и** служба (автозапуск), переключаемые тумблером «Автозапуск».

**Отступление (по твоей просьбе):**
- **Прямой запуск winws полностью убран.**
- **Старт** = установить службу (`start=auto`) с выбранной стратегией **и** запустить.
- **Стоп** = остановить **и удалить** службу.
- Тумблера «Автозапуск» больше нет — **сам Старт и есть автозапуск** (переживает перезагрузку).

**Последствия:**
- Любой Старт/Стоп = один UAC (элевация по требованию сохранена).
- Живой лог winws пропал (см. п.4).
- Поля/код прямого процесса (`child`, kill/wait, poll, Drop) вычищены.

## 7. Апдейтер ядра — несколько практических правок

Шаг 7 как в промпте (проверка релиза → скачивание → полная замена ядра), с отступлениями:

- **JSON GitHub API парсим вручную** (без `serde_json`) — нужны только `tag_name` и первый `.zip`. Легче по зависимостям.
- **Выгрузка драйвера перед заменой:** всплыл реальный баг — `os error 32` на `WinDivert64.sys`, потому что **драйвер WinDivert остаётся загружен** (служба `WinDivert` RUNNING) даже после остановки winws. Добавили `service::stop_driver()` (`net stop` + `sc delete` для `WinDivert`/`WinDivert14`) + паузу + ретраи копирования. (Промпт лишь в общем упоминал «файловые локи».)
- **Пользовательские списки** (`*-user.txt`) при замене сохраняются.
- **Предвыбор стратегии:** при старте читаем из реестра метку службы и подставляем её в дропдаун (чтобы показывалась реально запущенная стратегия, а не дефолт).

---

## 7-bis. Поставка БЕЗ ядра + кнопка «Скачать» (по твоему запросу)

Не описано в промптах.

- GUI ставится **без сборки Flowseal**. «Готовность ядра» = есть `version.txt` + ключевые файлы (`winws.exe`, `WinDivert.dll`, `WinDivert64.sys`) + распарсенные стратегии.
- Если ядра нет — вместо Старта показывается **«⤓ Скачать ядро»**, тянет актуальный релиз Flowseal и ставит в `core/` рядом с приложением.
- Скачивание пробует без прав, при непишемой папке — фолбэк на элевацию.

---

## 8. Настройки + трей — трей РЕАЛИЗОВАН, потом ПОЛНОСТЬЮ УБРАН (по твоему запросу)

Шаг 8 = персист настроек **и** системный трей (сворачивание, меню Старт/Стоп/Выход, «запускать свёрнутым»).

**Сохранение настроек — оставлено** (но уже без трея): стратегия (по имени) + Game Filter + IPSet в `%APPDATA%\Zaprust\config.json`.

**Трей — вырезан целиком** (был слишком багованным и не нужен):
- Удалены `tray.rs`, зависимость `tray-icon`, поток событий трея, скрытие/восстановление окна, опция «запускать свёрнутым».
- **По крестику окно просто закрывается** (дефолт eframe). Служба продолжает работать сама.

> То есть критерий шага 8 в части трея **сознательно не выполняется** — по твоему решению.

## Доп. инструментарий (вне промптов)

Добавлены скрытые CLI-режимы для диагностики и для элевированных операций:

| Флаг | Назначение |
|------|------------|
| `--dump-args "<стратегия>"` | показать итоговый argv winws |
| `--test-run "<стратегия>"` | поднять winws на 4с и показать его вывод |
| `--test-net` | детект WinDivert + прогон теста доменов |
| `--update-dry` | безопасно проверить апдейтер (во временную папку) |
| `--svc <install/remove/start/stop>` | элевированные операции со службой (реинвок) |
| `--apply-update <zip> <tag> …` | элевированная замена ядра |
